## 需求分析 AI 工作

1. 逐篇閱讀原稿，將每個疑似問題寫入 `content-review.md`。
2. 每項使用唯一 `TEXT-xxx`，包含文章、Word 段落、原文、建議、理由與狀態。
3. 依適合討論的批次與使用者確認，不一次要求使用者處理過多項目。
4. 未確認前狀態維持「待討論」，不得交由工程修改。
5. 使用者決議後，將狀態改為「同意修改」或「保留原文」。
6. 另外整理一份工程可執行清單，逐項提供：
   - `TEXT-xxx`
   - 文章 slug
   - 原文精確字串
   - 確認後字串
   - 是否同步修改 title、description、OG、Twitter 與 JSON-LD

## 工程回交條件

只有標示為「同意修改」的項目才交給網頁工程 AI。工程 AI 修改後應重新產製文章頁，並由網頁測試 AI確認：

- 同意修改項目已正確套用。
- 保留原文及待討論項目未被改寫。
- HTML 正文與 SEO metadata 沒有出現不同版本的文字。
